# CHME Cheer Phase 3 제품 확장 계획

## 목표
- 응원 경험 차별화
- 공개 인터랙션/뱃지/추천 고도화
- 운영 자동화와 대시보드 완성

## 핵심 과제
1. 뱃지 자동 부여
- sent 누적(50/100/500), scheduled 사용(20+) 조건 실시간 반영
2. 개인화 메시지
- 델타/시간대/관계 맥락 기반 추천 문구
3. 공개 인터랙션
- 마지막날 이후 응원자 확인 + 감사/리액션 + 피드 노출
4. 분석 대시보드
- 전환율(발송→열람→감사), 유지율, 시간대 성과

## KPI
- 감사 전환율
- 예약 응원 도달률
- 완주율 연동 uplift

## 완료 기준
- 뱃지 자동 부여 정확도 99%+
- 추천 메시지 클릭/사용률 목표 달성
- 공개 인터랙션 후 재참여율 상승

## 확정 의사결정 (반영 완료)
1. 추천 메시지 생성: **A안 채택**
- 규칙 기반 템플릿 사용 (델타/시간대 기반)
- 이유: 비용/지연/안전필터 운영 부담 대비 체감 효용이 높음

2. 공개 범위: **A안 채택**
- 챌린지 참여자 간에만 공개
- 이유: CHME의 비경쟁·저피로 커뮤니티 철학 유지

3. 뱃지 알림 타이밍: **혼합안 채택**
- 마일스톤성 뱃지: 즉시 토스트+푸시
- 누적/빈발 뱃지: 일일 요약
- 구현: badge 정책에 `notificationType: immediate | daily` 필드 적용

## 확인사항 (최종 확인 필요)
1. 템플릿 운영 방식
- 최적안 A: 템플릿 20~30개 + 델타 구간별 매핑(권장)
- 최적안 B: 시즌/이벤트 템플릿 세트 월 단위 교체

2. 공개 인터랙션 범위
- 최적안 A: 감사/리액션 2종으로 시작(권장)
- 최적안 B: 답글/멘션까지 동시 오픈

3. 알림 피로도 관리
- 최적안 A: 사용자별 일일 알림 상한 설정(권장)
- 최적안 B: 무상한 + 사용자 수동 끄기
