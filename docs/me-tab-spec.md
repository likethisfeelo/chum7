# ME 탭 구현 기획서

## 1) 목적
ME 탭에서 사용자가 **지금 무엇을 인증해야 하는지**를 가장 명확하게 보여주고, 챌린지 진행일(Day)과 섹션 분류가 실제 일정(시작일/오늘)과 일치하도록 한다.

---

## 2) 해결하려는 문제
1. Day 표시가 챌린지 시작일과 무관하게 `Day 1`로 고정되는 문제.
2. 상단 "지금 인증하기"가 오늘 기준 우선 인증 대상이 아닌 항목을 노출하는 문제.
3. 어제/이전일 미인증(백로그) 항목이 상단 인증 슬롯에 섞여 사용자 혼선을 만드는 문제.

---

## 3) 화면 구조(총 4개 섹션)
진행중 탭 기준으로 아래 순서를 유지한다.

### 섹션 1. 지금 인증하기 (인라인 인증 폼)
- 오늘 인증이 필요한 챌린지 중 **우선순위 1개**만 노출.
- 인라인 폼(텍스트/이미지/영상/링크)으로 바로 인증 가능.

### 섹션 2. 오늘 인증 예정(참여중 미인증 목록)
- 섹션 1에 선택되지 않은 나머지 미인증 항목 노출.
- 백로그가 있는 챌린지는 여기로 내려와야 함.

### 섹션 3. 인증 완료
- 오늘 인증 완료된 챌린지, 또는 7일 완주 이후 항목 노출.
- 카드 확장 시 Day 1~7 타임라인 확인 가능.

### 섹션 4. 빈 상태(Empty State)
- 진행중 챌린지가 없거나, 조건에 맞는 항목이 전혀 없을 때 안내 화면 노출.

---

## 4) 핵심 도메인 규칙

## 4.1 Day 계산 규칙
- 기본 원칙: `Day = (오늘 날짜 - 챌린지 시작일) + 1`
- 날짜만 비교(시/분/초 제거)한다.
- 시작일 파싱 실패 시 `currentDay`를 fallback으로 사용한다.
- 최소값은 1로 보정한다.

> 기대 효과: 시작일이 지난 챌린지는 Day 2, Day 3 ...으로 정확히 표시됨.

## 4.2 오늘 인증 완료 판정
해당 Day의 progress status가 아래 중 하나면 “오늘 인증 완료”로 본다.
- `success`
- `remedy`
- `failed` (정책상 해당 Day 인증 시도로 간주)

## 4.3 백로그(이전일 미완료) 판정
- 오늘 이전 Day(1 ~ today-1) 중 미완료 status가 하나라도 있으면 백로그로 판정.
- 백로그가 있는 챌린지는 **섹션 1 제외**, **섹션 2 배치**.

## 4.4 미인증 우선순위 정렬
미인증 항목은 개인 목표시각(`personalTarget`) 기준으로 정렬한다.
- 정렬값: `오늘 현재시각 → 다음 도래 목표시각까지 남은 분`
- 오름차순(가장 빨리 다가오는 항목 우선)
- 목표시각 미설정 시 기본 12:00(정오)로 처리

---

## 5) 섹션 배치 알고리즘

1. 진행중(active) 챌린지를 필터링한다.
2. 각 챌린지에 대해 `computedDay` 계산.
3. `computedDay <= 7` 이고 오늘 인증 미완료면 `unverified` 후보.
4. `unverified`를 “다음 도래 목표시각까지 남은 분” 오름차순 정렬.
5. `unverified` 중 **백로그 없는 첫 항목**을 섹션 1(`primary`)로 배치.
6. 섹션 2(`otherUnverified`)는 `unverified - primary` 전체를 노출.
7. 섹션 3(`verifiedToday`)는 아래 중 하나:
   - 오늘 인증 완료
   - `computedDay > 7` (완주 이후)

---

## 6) 인증 요청 파라미터 규칙
인라인 인증 제출 시 `day`는 `currentDay`가 아니라 `computedDay`를 사용한다.
- UI Day 표시값과 서버 저장 Day를 일치시켜 이력 왜곡을 방지.

---

## 7) 예시 시나리오

### 시나리오 A
- 챌린지 A: 시작일 3일 전, 오늘 Day 4, 백로그 없음, 목표 07:00
- 챌린지 B: 시작일 5일 전, 오늘 Day 6, 어제 미인증(백로그 있음), 목표 06:30
- 챌린지 C: 오늘 Day 2, 백로그 없음, 목표 08:00

정렬 결과(시간 기준): B(06:30), A(07:00), C(08:00)
- 섹션 1: A (B는 백로그로 제외, A가 백로그 없는 첫 항목)
- 섹션 2: B, C
- 섹션 3: 오늘 완료 항목만

### 시나리오 B
- 모든 미인증 항목에 백로그가 있음
- 섹션 1: 없음
- 섹션 2: 미인증 전체 노출

---

## 8) 비기능/품질 기준
- 시간 계산은 로컬 타임존 기준으로 일관 처리.
- Day 계산 로직은 화면 컴포넌트 간 동일 규칙 사용.
- 리스트 정렬/분류는 렌더마다 흔들리지 않도록 순수 계산(useMemo) 유지.

---

## 9) QA 체크리스트
1. 시작일이 어제인 챌린지가 Day 2로 표시되는가?
2. 시작일이 5일 전이면 Day 6으로 표시되는가?
3. 백로그가 있는 챌린지가 섹션 1에 노출되지 않는가?
4. 미인증 목록이 "가장 빠르게 도래하는 목표시각" 순으로 정렬되는가?
5. 인증 제출 payload의 `day`가 화면 표시 Day와 일치하는가?
6. 오늘 완료 처리(`success/remedy/failed`)가 섹션 3으로 이동하는가?

---

## 10) 향후 개선안
- "오늘 인증 예정" 섹션명을 "참여중 챌린지"와 병기하여 의미 명확화.
- 백로그 항목에 `미인증 N일` 뱃지 추가.
- 서버에서 `computedDay`를 내려주도록 API 확장해 프론트 계산 의존도 축소.
