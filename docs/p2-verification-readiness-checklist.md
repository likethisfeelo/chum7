# P2 검증/운영준비 체크리스트 (후속 진행용)

> 상태: 보류(사용자 요청에 따라 문서화만 진행)

## 현재 릴리스 정책
- P1 생성/참여 흐름은 먼저 배포하여 라이브 스모크 테스트를 진행한다.
- P2/Phase5 항목은 본 문서 체크리스트로만 유지하고, 실행은 후속 스프린트에서 진행한다.
- P1 배포 준비/스모크 절차는 `docs/p1-layer-flow-live-test-readiness.md`를 기준으로 수행한다.

## 1. E2E 회귀 시나리오
- 신규 참여자 가입 → 챌린지 참여 → Day 인증 완료.
- 실패 Day 발생 → Day 6 remedy 제출 → 점수/진행도 반영.
- 추가 인증 생성(`isExtra`) → 개인 보관 → 공개 전환.
- 관리자 제출물 승인/반려/당일 정정 → 감사로그 반영.

## 2. 부하/성능 검증
- 피드 조회 집중 시간대 `GET /verifications?isPublic=true` 부하 테스트.
- Day6 remedy 집중 시간대 `POST /verifications/remedy` 부하 테스트.
- 운영 대시보드/감사로그 API 동시 조회 부하 테스트.

## 3. 데이터 정합성 검증
- `practiceAt`, `uploadAt`, `certDate`, `day` 계산 정합성 표본 점검.
- `isExtra`, `isPersonalOnly`, `primaryVerificationId` 연결성 점검.
- 공개 전환 후 피드 반영 지연 시간 측정.

## 4. 운영 런북/롤백 준비
- 장애 유형별 즉시 대응 플로우(알림 실패/승인 지연/잘못된 공개 전환).
- 테이블/GSI 변경 시 롤백 및 재처리 절차 점검.
- 최종 출시 체크리스트 100% 충족 후 승격.
