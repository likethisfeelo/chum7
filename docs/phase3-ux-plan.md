# Phase 3 UX 고도화 계획 (시나리오 분기 중심)

## 목적
- PHASE 1~2에서 확보한 공통 기반/플로우 A~D를 바탕으로, **챌린지 타입 × 레메디 정책** 조합별 UX를 차별화한다.
- 사용자가 "지금 내 챌린지에서 무엇을 해야 하는지"를 더 빠르게 인지하게 만든다.

---

## 1) PHASE 2 대비 PHASE 3의 핵심 차이

| 구분 | PHASE 2 (현재) | PHASE 3 (목표) |
|---|---|---|
| UX 구조 | Flow A~D 공통 테스트 중심 | 시나리오별 맞춤 가이드/강조 |
| 정보 제공 | 정책/상태를 동일 포맷으로 노출 | 타입/정책별 문구·컴포넌트·우선순위 다르게 노출 |
| 사용자 인지 | "기능이 된다" 검증 | "내 상황에 딱 맞다" 체감 강화 |
| 테스트 기준 | 플로우 완료 가능 여부 | 시나리오별 행동 유도 성능(오해/이탈 감소) |

---

## 2) 시나리오별 UX 차이 설계

### S1. `leader_only + strict` (엄격 완주형)

- **UX 목표**: 실패 허용 없음 메시지를 명확히 전달해 기대치 불일치 방지.
- **핵심 차이**
  - 레메디 메뉴/진입 버튼 완전 숨김.
  - Day 4~5에 "이 챌린지는 보완 불가" 경고 배너 고정 노출.
  - 인증 누락 리스크를 강조하는 카피(예: "남은 N일 모두 인증 필요").
- **주요 화면**: `/me`, `VerificationSheet`

### S2. `leader_only + limited` (리더 가이드형)

- **UX 목표**: 제한 보완 기회를 미리 인지시켜 완주 가능성 극대화.
- **핵심 차이**
  - "남은 보완 X회" 상시 노출.
  - Day 4~5 예고 카드(“Day 6 보완 준비”).
  - 보완 횟수 소진 시 즉시 경고 상태로 전환.
- **주요 화면**: `/me`, `/verification/remedy`

### S3. `personal_only + open` (자기설계 자유형)

- **UX 목표**: 개인 목표 중심 경험 강화.
- **핵심 차이**
  - 개인 목표/목표시각을 카드 상단 고정.
  - delta 피드백을 “내 목표 대비” 문장으로 강조.
  - 개인 퀘스트 승인 상태를 오늘 할 일 위에 표시.
- **주요 화면**: `/me`, `VerificationSheet`

### S4. `leader_personal + limited` (구조적 병행형)

- **UX 목표**: 공통/개인 퀘스트 병행 시 혼란 최소화.
- **핵심 차이**
  - 개인 퀘스트 상태칩 + 다음 액션 버튼(수정/대기/완료) 강조.
  - 공통/개인 오늘 할 일을 2열로 분리 표시.
  - 반려 시 재제출 deadline/가이드 노출.
- **주요 화면**: `/me`, `/quests`, `/challenges/:id`

### S5. `mixed + open` (멀티레이어 통합형)

- **UX 목표**: 레이어별 복잡도를 구조화해 피로감 완화.
- **핵심 차이**
  - A/B/D 레이어 섹션 분리.
  - 레이어별 진행률 미니 게이지.
  - 오늘 인증 우선순위 추천(예: 미완료 레이어 우선).
- **주요 화면**: `/quests`, `/me`

## 3) 플로우/기능 변경 매핑 (A~D)

| 플로우 | PHASE 3에서 추가할 것 | 기능 단위 |
|---|---|---|
| Flow A (참여) | 타입별 기대치 안내 강화 | 챌린지 상세 상단 가이드 카드 분기 |
| Flow B (개인퀘스트) | 상태별 다음 액션 고도화 | 상태칩 + CTA + 데드라인/피드백 블록 |
| Flow C (일일 인증) | 시나리오별 피드백 문구 차별화 | 성공 토스트/요약 카드 템플릿 분기 |
| Flow D (레메디) | 정책별 노출전략 세분화 | strict 숨김, limited 카운트다운, open 유연 안내 |

## 4) 구현 우선순위 (권장)

### P3-1 (1주)
1. 시나리오 판별 규칙 유틸 추가 (`challengeType + remedyPolicy + personalQuestEnabled`).
2. `/me` 카드에서 strict/limited/open 분기 UI 적용.
3. 개인퀘스트 상태칩 액션 강화(rejected/revision_pending 대응).

### P3-2 (1주)
1. `/quests` 레이어 섹션 분리(A/B/D).
2. 레이어별 완료율 표시 + 우선 수행 추천.
3. VerificationSheet 피드백 문구 시나리오 분기.

### P3-3 (0.5~1주)
1. 카피라이팅/아이콘/배너 톤 조정.
2. QA 시나리오 5종 회귀 테스트.
3. 측정 지표 점검(이탈률, 인증 완료율, 레메디 사용률).

---

## 5) QA 체크 포인트 (PHASE 3)

- [ ] strict 챌린지에서 레메디 진입 버튼이 어디에도 노출되지 않는다.
- [ ] limited 챌린지에서 남은 보완 횟수가 화면 전반에서 일관된다.
- [ ] personal_only 챌린지에서 개인 목표 정보가 우선순위 높은 위치에 표시된다.
- [ ] leader_personal에서 개인퀘스트 상태별 다음 액션이 즉시 이해된다.
- [ ] mixed 챌린지에서 A/B/D 레이어 진행률이 각각 확인 가능하다.
