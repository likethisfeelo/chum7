# ADR-0001: Challenge-Quest 정책 기준선 확정

- 상태: Accepted
- 날짜: 2026-02
- 범위: Challenge-Quest V2 (Admin-First)

## 문맥

챌린지·퀘스트는 Layer A/B/D, Remedy Policy, Day 계산, 승인 워크플로우가 결합된 복합 정책 시스템이다.
정책이 UI보다 늦게 확정되면 운영 중 정책 충돌/데이터 정합성 이슈가 커진다.

## 결정

1. 타임존 기준은 KST(Asia/Seoul)로 정의한다.
2. 데이터 저장 시각은 UTC로 저장한다.
3. Day 계산은 사용자 로컬 타임존 경계(00:00~23:59:59)를 기준으로 산정한다.
4. 완주 조건은 아래를 따른다.
   - Layer A/B: Remedy 포함 100% 달성 시 완주
   - Layer D: 기간 내 미수행은 Remedy 불가, 미완료 시 완주 실패
5. 리더 퀘스트 수정은 진행 중 원칙적 금지, 오타/표현 정정만 허용한다.
6. Layer B 승인 정책은 자동승인 기본, 관리자 수동승인 전환 가능 구조를 채택한다.
7. 수동승인 모드에서 관리자 지정 보조승인자에게 승인 권한을 위임할 수 있다.
8. 승인/반려 결과는 당일 내 정정(취소/반려/재승인) 가능하며 모든 변경은 감사로그에 남긴다.
9. 운영 SLA 수치 목표는 현 단계에서 고정하지 않고 운영팀 합의 후 별도 ADR로 확정한다.

## 근거

- 글로벌 런칭 대비를 위해 저장(UTC)과 사용자 체감 Day(로컬 경계)를 분리해야 한다.
- 승인 실수 복구 기능이 없으면 운영 리스크가 급증하며 CS 비용이 커진다.
- 정책 고정과 로그 강화가 선행되면 참여자 UX 개발 시 재작업 비용을 크게 줄일 수 있다.

## 결과

- Phase 0 산출물(OpenAPI 초안/상태전이표)을 본 ADR 기준으로 유지한다.
- 구현 단계에서 정책 충돌 발생 시 본 ADR을 우선 규범으로 판단한다.
